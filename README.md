## Clinical Data Quality & EDC Query Management Dashboard

## Project Overview

This project simulates a simplified Clinical Data Management (CDM) workflow based on Electronic Data Capture (EDC) systems used in clinical trials.

The objective is to explore how clinical data inconsistencies are identified, tracked, and resolved through a query lifecycle, as well as how subject-level data can be reviewed for quality and consistency.

The project reflects real-world practices in Clinical Research Organizations (CROs), pharmaceutical environments, and Real-World Data (RWD) workflows.

---

## Objectives

- Explore clinical data quality issues such as missing values, inconsistencies, and implausible measurements  
- Simulate a query management workflow (Open → Answered → Closed)  
- Analyze discrepancies across clinical domains (demographics, vital signs)  
- Enable subject-level data exploration and review  
- Build structured dashboards for clinical data quality monitoring  

---

## Dataset Description

The project uses two structured datasets:

### Query Management Dataset (EDC Queries)

This dataset simulates data queries generated during clinical data review.

It includes:
- Field-level discrepancies
- Query status tracking
- Communication between data managers and site users
- Timestamps for query lifecycle events

Typical query statuses:
- Open
- Answered
- Closed

---

### Subject-Level Clinical Dataset

This dataset contains patient-level clinical data including:

- Demographics (age, sex, ethnicity, country)
- Vital signs (height, weight, BMI, blood pressure, heart rate, temperature, oxygen saturation)

---

## Dashboards

### EDC Data Overview Dashboard

Provides a global view of the dataset structure and completeness, including:

- Distribution of clinical fields
- Overview of data completeness
- High-level dataset structure

---

### Discrepancies Dashboard

Focuses on clinical data quality issues:

- Missing values detection
- Implausible or inconsistent values
- Cross-field validation issues (e.g. BMI inconsistencies)
- Query status distribution (Open / Answered / Closed)

---

### Subject Drilldown Dashboard

Enables detailed subject-level exploration:

- Individual patient clinical profiles
- Field-level values per subject
- Associated discrepancies and queries
- Cross-subject comparison

## Data Quality Considerations

The analysis addresses common clinical data issues such as:

- Missing demographic or clinical values  
- Implausible physiological measurements  
- Logical inconsistencies between related variables (e.g. BMI vs height/weight)  
- Duplicate or redundant entries  
- Query resolution delays and workflow inefficiencies
