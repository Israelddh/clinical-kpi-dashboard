# Clinical Data Quality & EDC Query Management Dashboard

## Project Overview

This project simulates a simplified Clinical Data Management (CDM) workflow based on Electronic Data Capture (EDC) systems used in clinical trials.

The objective is to explore how clinical data inconsistencies are identified, tracked, and managed through a query lifecycle, as well as how subject-level data can be reviewed for quality and consistency.

The project is designed to reflect real-world practices in clinical data environments (CRO / pharma / Real World Data contexts).

---

## Objectives

The main objectives of this project are:

- Explore clinical data quality issues (missing values, inconsistencies, outliers)
- Simulate a query management workflow (Open → Answered → Closed)
- Analyze discrepancies across clinical domains (demographics, vital signs)
- Provide subject-level data exploration and drill-down capability
- Build simple analytical dashboards for data review

---

## Dataset Description

The project uses two structured datasets:

### 1. Query Management Dataset (EDC Queries)

This dataset simulates data queries generated during clinical data review.

It includes:

- Field-level discrepancies
- Query status tracking
- Message logs between data manager and site users
- Timestamps for query lifecycle events

Typical statuses include:
- Open
- Answered
- Closed

---

### 2. Subject-Level Clinical Dataset

This dataset contains structured patient-level data including:

- Demographics (age, sex, ethnicity, country)
- Vital signs (height, weight, BMI, blood pressure, heart rate, temperature, oxygen saturation)

---

## Dashboards Developed

### 1. EDC Data Overview Dashboard

Provides a global view of the clinical dataset, including:

- Distribution of available fields
- Overview of subject-level completeness
- General structure of collected clinical data

---

### 2. Discrepancies Dashboard

Focuses on data quality issues detected in the dataset.

Includes:

- Missing values identification
- Implausible or inconsistent values
- Cross-field validation issues (e.g., BMI inconsistencies)
- Query status distribution (Open / Answered / Closed)

---

### 3. Subject Drilldown Dashboard

Allows detailed exploration at subject level.

Includes:

- Individual subject clinical profiles
- Field-level values per subject
- Associated discrepancies or queries (if present)
- Comparison across subjects

## Data Quality Considerations

During analysis, common types of issues observed in clinical datasets are considered, such as:

- Missing demographic or clinical values
- Implausible physiological measurements
- Logical inconsistencies between related variables (e.g., BMI vs height/weight)
- Duplicate or redundant entries
- Delayed query resolution workflows
